# ChatOutSafe

很多人想要导出聊天记录来‘蒸馏’，但苦于破解数据库复杂、不安全和怕封号

那这时候就有人要问了：主播主播，有没有更安全的方法

有的，兄弟，有的

那就要介绍一下这个小项目了

很简单，就是把微信/QQ聊天记录逐屏截下来，按顺序存成图片，然后交给有识图功能的AI，让他给你保存成json等文件

原理很简单：定位到聊天窗口的消息区域，模拟鼠标滚轮往下滚，每滚一屏截一张图，滚到底了自动停。滚动节奏做了随机化处理，不是那种机械式的匀速滚，尽量像人在操作。

## 环境要求

- Windows 10/11
- Python 3.8+

## 从零开始

### 1. 把仓库拉下来

```bash
git clone https://github.com/Githao3/chat-out-safe.git
cd chat-out-safe
```

### 2. 装依赖

建议先建个虚拟环境，不建也行：

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

依赖就四个：`pywin32`（调 Windows API）、`Pillow`（截图）、`pyautogui`（模拟键鼠）、`numpy`（算图片相似度）。

### 3. 开始截图

**截微信聊天：**

先打开微信，进入你想截的那个聊天窗口，然后：

```bash
python wechat_capture.py
```

程序会自动找到微信窗口，把窗口置顶，倒数 3 秒后开始从当前位置往下逐屏截。截完的图存在 `./output_wechat/` 里，命名是 `page_001.png`、`page_002.png` 这样。

**截QQ聊天：**

同理，先打开QQ进到目标聊天：

```bash
python qq_capture.py
```

截图存在 `./output_qq/`。

### 4. 几个实用的用法

```bash
# 先滚到聊天顶部（最早的消息），再从头开始截
python wechat_capture.py --scroll-top

# 只截 10 张试试效果
python wechat_capture.py --max-pages 10

# 换个输出目录
python wechat_capture.py --output-dir D:\聊天备份\张三

# 滚动量调大一点，适合消息比较长的群聊
python wechat_capture.py --scroll-amount 300
```

## 参数一览

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--output-dir` | `./output_wechat` / `./output_qq` | 截图保存到哪 |
| `--scroll-amount` | `200` | 每次滚轮滚多远，值越大单次滚动越多 |
| `--scroll-delay` | `1.5` | 滚完等多久再截图（秒） |
| `--similarity-threshold` | `0.98` | 两张图有多像算"没动过"，0~1 之间 |
| `--max-pages` | `1000` | 最多截多少张，防止跑飞 |
| `--offset-left` | `280` / `260` | 聊天区域左边往里收多少像素（去掉侧边栏） |
| `--offset-top` | `65` / `110` | 聊天区域上边往里收多少像素（去掉标题栏） |
| `--offset-bottom` | `120` / `80` | 聊天区域下边往里收多少像素（去掉输入框） |
| `--scroll-top` | 关 | 加上这个参数会先滚到顶部再截 |

## 截图区域是怎么算的

程序拿到微信/QQ窗口的位置后，会把四周裁掉一部分，只留中间的消息区域：

```
┌──────────────────────────┐
│  裁掉 (offset-top)        │
│  ┌────────────────────┐  │
│  │                    │  │
│裁│   聊天消息区域      │裁│
│掉│   （实际截图部分）   │掉│
│  │                    │  │
│  └────────────────────┘  │
│  裁掉 (offset-bottom)     │
└──────────────────────────┘
    ↑ 裁掉 (offset-left)
```

不同版本的微信/QQ布局不一样，如果截出来的图位置不对，调一下 `--offset-*` 参数就行。可以先用 `--max-pages 1` 截一张看看效果。

## 怎么停

- **自动停**：滚到底了，连续两次截图几乎一样，程序自动结束
- **手动停**：随时按 `Ctrl+C`，已经截好的图不会丢
- **紧急停**：把鼠标甩到屏幕左上角，pyautogui 的 FailSafe 机制会立即中断

## 注意事项

- 截图用的是屏幕坐标，截图期间别拖动或缩放窗口
- 建议先用 `--max-pages 3` 试截几张，确认区域没问题再批量跑
- 不同显示器缩放比例（100%/125%/150%）下偏移量可能不同，需要微调
- 程序会在截图区域中心附近随机位置点击来获取焦点，确保鼠标不要挡住聊天内容
